[中国] 的论文[《Externalization in LLM Agents: A Unified Review of Memory, Skills, Protocols and Harness Engineering》][paper1-url]，由上海交通大学、中山大学、CMU 等机构的研究者合作完成，提出一个统一框架来理解 LLM Agent 的外化设计范式。

过去几年，LLM Agent 的设计思路发生了根本性转变：从依赖模型内部能力，转向将能力外化为可管理的组件。这篇综述系统梳理了这一转变，提出了记忆、技能、协议、框架四维度外化框架。

## 为什么需要外化

传统 AI 系统依赖模型权重来编码所有能力。但 LLM Agent 需要处理复杂、动态的任务，固定权重无法满足。研究者发现，将能力从模型内部转移到外部，可以获得更大的可靠性提升：

- **记忆外化**：用外部存储替代模型内部记忆
- **技能外化**：将可复用能力封装为独立模块
- **协议外化**：标准化模块间的交互方式
- **框架外化**：统一管理上述组件的运行时环境

这个思路与人类认知外化的历史高度相似——从思维到语言，到文字，到印刷，再到数字计算，每一次认知能力的提升都伴随着外化的突破。

## 四维度框架

论文提出的框架将 Agent 设计空间划分为四个层次：

**记忆（Memory）**：包括短期记忆、长期记忆和外部知识库。关键设计决策是存储什么、何时存储、如何检索。

**技能（Skills）**：将特定能力（如代码生成、数学推理）封装为可复用的外部模块，按需调用。

**协议（Protocols）**：定义 Agent 内部各组件之间的交互规则，包括消息格式、调用顺序、错误处理等。

**框架（Harness）**：统一调度上述组件的运行时环境，确保它们协同工作。

## 对行业的影响

这篇综述的价值在于提供了一个**统一的设计语言**。过去 Agent 设计各自为战，缺乏系统性分析。外化框架让研究者可以清晰地描述和比较不同 Agent 架构的优劣。

对工程实践而言，它明确了 Agent 设计的核心问题：**哪些能力应该留在模型内部，哪些应该外化**。这个判断直接影响系统的可靠性、可扩展性和维护成本。
